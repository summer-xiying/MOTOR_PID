# VOFA+ 在线调参与 PID 参数说明

## 1. 通信协议

### 1.1 VOFA+ JustFloat 波形协议

- **帧格式**：N × float(4字节) + 帧尾 `0x00 0x00 0x80 0x7F`
- **发送周期**：100ms（定时器中断 10ms，每 10 次发送一次）
- **触发条件**：`debug_output_enabled = 1`

### 1.2 VOFA+ 文本命令协议（滑块组件）

格式：`name:value\n`（换行符结尾，VOFA+ 中配置滑块组件名即可）

#### 速度环 PID

| 滑块名称 | 作用 | 值范围 | 实际效果 |
|----------|------|--------|----------|
| `kp` | 速度环比例系数 | ≥ 0 | 同时写入 kp_1 和 kp_2 |
| `ki` | 速度环积分系数 | ≥ 0 | 同时写入 ki_1 和 ki_2 |
| `kd` | 速度环微分系数 | ≥ 0 | 同时写入 kd_1 和 kd_2 |

#### 循迹环 PD

| 滑块名称 | 作用 | 值范围 | 实际效果 |
|----------|------|--------|----------|
| `kp_track` | 循迹比例系数 | ≥ 0 | 统一参数，不分直线/弯道 |
| `kd_track` | 循迹微分系数 | ≥ 0 | 统一参数，不分直线/弯道 |

#### 控制开关与速度

| 滑块名称 | 作用 | 值范围 | 实际效果 |
|----------|------|--------|----------|
| `speed` | 目标速度 | 0 ~ 500 | 同时设置两电机，**会关闭循迹** |
| `track` | 循迹开关 | 0 / 1 | 1 = 开启循迹，0 = 关闭 |
| `output` | 波形输出开关 | 0 / 1 | 1 = 发送波形，0 = 静默 |

#### 示例

```
kp:0.8\n        → 速度环 Kp = 0.8（两电机）
kp_track:15.0\n → 循迹 Kp = 15.0
speed:150\n     → 两电机目标速度 150 mm/s，关闭循迹
track:1\n       → 开启循迹
```

### 1.3 WirelessTune 二进制协议

- **帧头**：`0xAA 0x55`
- **帧格式**：`[HEAD1][HEAD2][CMD][LEN][DATA...][CHECK]`
- **校验**：CHECK = CMD + LEN + SUM(DATA)

| CMD | 功能 | DATA 长度 | 说明 |
|-----|------|-----------|------|
| 0x01 | 设置速度环 PID | 13 字节 | motor_id(1) + kp(4) + ki(4) + kd(4)，支持单独设置某电机 |
| 0x02 | 设置速度 | 8 字节 | left_speed(4) + right_speed(4)，会关闭循迹 |
| 0x03 | 查询状态 | 0 字节 | 返回 54 字节状态帧 |
| 0x04 | 波形输出开关 | 1 字节 | 0 = 关，1 = 开 |
| 0x05 | 循迹开关 | 1 字节 | 0 = 关，1 = 开 |

## 2. VOFA+ 波形通道映射

函数 `VOFA_SendPIDWaveform()`（`uart.c`），6 通道：

| 通道 | 变量 | 含义 |
|------|------|------|
| CH0 | `target_speed_1` | 电机 1 目标速度 (mm/s) |
| CH1 | `speed_1` | 电机 1 实际速度 (mm/s) |
| CH2 | `PWM_1_duty` | 电机 1 PWM 占空比 (0~4000) |
| CH3 | `target_speed_2` | 电机 2 目标速度 (mm/s) |
| CH4 | `speed_2` | 电机 2 实际速度 (mm/s) |
| CH5 | `PWM_2_duty` | 电机 2 PWM 占空比 (0~4000) |

## 3. 速度环 PID 参数

### 3.1 当前参数：第二组数据（2026-06-11）

| 变量 | 电机 1 | 电机 2 | 说明 |
|------|--------|--------|------|
| `kp` | 2.0 | 2.0 | 比例系数 |
| `ki` | 0.15 | 0.15 | 积分系数 |
| `kd` | 2.5 | 2.5 | 微分系数 |

> 第一组数据（2026-06-10）：kp=0.8, ki=0.3, kd=0.12

- VOFA 命令 `kp`/`ki`/`kd` 同时写入两组参数
- WirelessTune 二进制协议（CMD 0x01）可通过 motor_id 单独设置

### 3.2 内部常量（`pid.c` #define）

| 常量 | 值 | 说明 |
|------|-----|------|
| `ERROR_THRESHOLD` | 30 | 积分限幅，单次最大积分增量 |
| `PWM_DEADZONE` | 500 | 死区补偿阈值 |
| `DEADZONE_SPEED_THRESH` | 50.0 | 目标速度高于此值时才启用死区补偿 |
| `SPEED_FILTER_ALPHA` | 0.35 | 速度测量低通滤波系数，越小越平滑 |
| `D_FILTER_ALPHA` | 0.5 | D 项低通滤波系数，越小越平滑 |
| PWM 上限 | 4000 | 硬件限制 |

### 3.3 堵转保护

- 条件：PWM ≥ 3500 且速度 < 15 mm/s，持续 15 周期（150ms）
- 动作：PWM 清零，冷却 50 周期（500ms）

## 4. 循迹环参数（`pid.c`）

### 4.1 循迹 PD 参数（统一，不分直线/弯道）

| 变量 | 初始值 | VOFA 命令 | 说明 |
|------|--------|-----------|------|
| `kp_track` | 12.0 | `kp_track` | 比例系数 |
| `kd_track` | 50.0 | `kd_track` | 微分系数 |

> 循迹 PD 输出 = kp_track × position + kd_track × d(position)/dt

### 4.2 速度策略参数（#define）

| 参数 | 值 | 说明 |
|------|-----|------|
| `BASE_SPEED` | 170 | 基础速度 (mm/s) |
| `MODE_SWITCH_THRESH` | 2.5 | 弯道判定阈值（\|position\| ≥ 此值视为弯道） |
| `OUTER_TURN_GAIN` | 1.45 | 外轮增益（弯道中外轮加速） |
| `INNER_TURN_GAIN` | 0.30 | 内轮增益（弯道中内轮减速） |
| `TARGET_SPEED_STEP_LIMIT` | 12.0 | 直线斜坡限制 / 10ms |
| 弯道斜坡 | 24.0 | 弯道模式斜坡限制 |
| 边缘救线斜坡 | 32.0 | 边缘传感器触发时斜坡限制 |

### 4.3 弯道双基准速度

| 参数 | 公式 | 下限 |
|------|------|------|
| 内轮因子 | `1.0 - 0.05 × |position|` | 0.7 |
| 外轮因子 | `1.0 - 0.025 × |position|` | 0.85 |

### 4.4 差速补偿（仅弯道判定为直线时生效）

| 参数 | 值 | 说明 |
|------|-----|------|
| `DIFF_KP` | 0.5 | 差速比例 |
| `DIFF_KI` | 0.1 | 差速积分 |
| `DIFF_KD` | 0.05 | 差速微分 |
| `DIFF_LIMIT` | 30.0 | 补偿限幅 |
